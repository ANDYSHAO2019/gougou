# 汪汪大作战项目状态

## 当前版本重点

- 卡通后院背景已接入。
- 玩家和 AI 双狗推线对战已完成。
- AI 会随机播放 3 段真实狗叫声。
- 玩家麦克风输入已从单纯音量触发升级为规则型狗叫检测。
- 开局会做环境噪声校准和试叫校准。
- 选狗界面已升级为图片卡片、属性条、解锁状态。
- 狗狗属性已影响玩法：力量、耐力、宽容、暴击。
- 胜负结算有 K.O. 震屏、胜利弹跳、失败趴下、奖励 WP。
- 连胜奖励和存档已接入。
- 资源预加载和 smoke test 已接入。
- 联网对战已简化为房间号模式：房主创建房间，朋友输入房间号加入，页面会自动交换 WebRTC 信令。

## 备份说明

备份目录在 `backups/` 下，按阶段保存：

- `v001_baseline_*`：开始本轮任务前的可运行基线。
- `v002_calibration_*`：加入麦克风校准后的版本。
- `v003_select_growth_*`：升级选狗和狗狗属性后的版本。
- `v004_audio_rewards_*`：加入音效、连胜奖励后的版本。
- `v005_preload_smoketest_*`：加入资源预加载和 smoke test 后的版本。
- `v007_before_online_*`：加入联网前的可运行版本。
- `v008_online_p2p_*`：加入 WebRTC P2P 联网对战后的版本。
- `v011_peerjs_room_online_*`：把联网对战简化为房间号模式后的版本。
- `v012_ntfy_room_online_*`：使用房间号和 ntfy 自动信令后的版本。

如果新版本出问题，可以把对应备份里的 `index.html`、`styles.css`、`game.js` 和 `assets/` 复制回项目根目录。

## 本地测试

启动/访问：

```bash
http://127.0.0.1:5173/
```

静态检查：

```bash
node tools/smoke-test.js
```

重点验收场景：

- 正常进入选狗界面。
- 选择/解锁狗狗。
- 点击开始后先出现“安静”和“试叫”校准。
- 模仿短促“汪”能触发攻击。
- 长时间大喊会提示“拖太长了”或不触发。
- 拍桌子会提示“敲击太短”或不触发。
- AI 会随机播放真实狗叫。
- 玩家胜利、失败、平局都能正常结算。
- 刷新后 WP、解锁和连胜状态能读取。
- 联网对战验收：
  - A 端点击“联网对战” -> “创建房间”，复制“房间号”发给 B。
  - B 端点击“联网对战”，输入 A 的房间号，点击“加入房间”。
  - 两边状态显示“已连接”后，进入战斗。
  - 双方都点击开始，完成校准后由房主同步倒计时。
  - 任意一方有效狗叫，另一方应看到右侧狗攻击、自己受击、推线变化。
